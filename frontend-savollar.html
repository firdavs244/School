1useState hook nima uchun ishlatiladi? +

useState - React komponentida o'zgaruvchan ma'lumotlarni saqlash uchun ishlatiladi.

Loyihamizdagi misol (LoginPage.jsx):

const [loading, setLoading] = useState(false);

// loading - hozirgi qiymat
// setLoading - qiymatni o'zgartirish funksiyasi
// false - boshlang'ich qiymat
useState qiymati o'zgarganda, komponent qayta renderlanadi (ekranda yangilanadi).
2useEffect hook nima uchun ishlatiladi? +

useEffect - komponent yuklanganda yoki ma'lum qiymatlar o'zgarganda ishlaydigan kod yozish uchun.

Loyihamizdagi misol (ProtectedRoute.jsx):

useEffect(() => {
  const verifyToken = async () => {
    // Token tekshirish...
  };
  verifyToken();
}, []); // [] - faqat 1 marta ishlaydi (komponent yuklanganda)

Dependency array:

  • [] - faqat 1 marta (mount'da)
  • [value] - value o'zgarganda
  • Bo'sh (yozilmagan) - har renderda
3props nima va qanday ishlatiladi? +

props - ota komponentdan bola komponentga ma'lumot uzatish usuli.

Loyihamizdagi misol (ProtectedRoute.jsx):

// Ota komponent (App.jsx):
<ProtectedRoute>
  <Dashboard />
</ProtectedRoute>

// Bola komponent (ProtectedRoute.jsx):
const ProtectedRoute = ({ children }) => {
  // children = <Dashboard />
  return children;
};
props faqat o'qish uchun - o'zgartirish mumkin emas!
4Komponent nima? Funksional va Class komponentlar farqi? +

Komponent - UI'ning qayta ishlatiladigan qismi. Har bir sahifa, tugma, forma - barchasi komponent.

Funksional komponent (hozirgi standart):

function LoginPage() {
  const [loading, setLoading] = useState(false);
  return <div>Login</div>;
}

Class komponent (eski usul):

class LoginPage extends React.Component {
  state = { loading: false };
  render() {
    return <div>Login</div>;
  }
}
Loyihamizda faqat funksional komponentlar ishlatilgan - bu zamonaviy standart!

// Bu JSX:

Muhim farqlar HTML dan:

  • class ‚Üí className
  • for ‚Üí htmlFor
  • JavaScript: {qiymat} qavslar ichida

7Routes va Route qanday ishlaydi? +

Routes - barcha Route'larni o'z ichiga oladi.

Route - URL va komponent bog'lanishi.

<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/login" element={<LoginPage />} />
  <Route path="/dashboard" element={<Dashboard />} />

URL /login bo'lsa ‚Üí LoginPage ko'rsatiladi.

8useNavigate hook nima qiladi? +

useNavigate - JavaScript orqali boshqa sahifaga o'tish.

import { useNavigate } from 'react-router-dom';

function LoginPage() {
  const navigate = useNavigate();
  
  const handleSubmit = async () => {
    // Login muvaffaqiyatli bo'lganda:
    navigate('/dashboard'); // Dashboard'ga o'tish
  };

9Link va Navigate farqi nima? +

Link - foydalanuvchi bosganda o'tish (a tag o'rnida):

<Link to="/register">Ro'yxatdan o'tish</Link>

Navigate - avtomatik o'tish (redirect):

// ProtectedRoute.jsx da:
if (!isValid) {
  return <Navigate to="/login" replace />;
}
Link = foydalanuvchi tanlaydi, Navigate = avtomatik yo'naltirish
10ProtectedRoute qanday ishlaydi? +

ProtectedRoute - faqat tizimga kirgan foydalanuvchilarga ruxsat beradi.

const ProtectedRoute = ({ children }) => {
  // 1. Token bormi tekshirish
  // 2. Token haqiqiymi tekshirish (API so'rov)
  
  if (!isValid) {
    return <Navigate to="/login" />; // Login'ga yuborish
  }
  
  return children; // Sahifani ko'rsatish
};

Ishlatilishi:

<Route path="/dashboard" element={
  <ProtectedRoute>
    <Dashboard />
  </ProtectedRoute>
} />

üîå API va Asinxron JavaScript

11async/await nima? +

async/await - asinxron kodni sinxron ko'rinishda yozish.

// Eski usul (Promise):
loginUser(email, password)
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await:
const handleSubmit = async () => {
  try {
    const data = await loginUser(email, password);
    console.log(data);
  } catch (err) {
    console.error(err);
  }
};
await - Promise kutish uchun, faqat async funksiya ichida ishlaydi.
12fetch API qanday ishlaydi? +

fetch - serverga HTTP so'rov yuborish.

// GET so'rov:
const response = await fetch('https://api.example.com/users');
const data = await response.json();

// POST so'rov:
const response = await fetch('https://api.example.com/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password })
});
const data = await response.json();
13try/catch nima uchun kerak? +

try/catch - xatolarni ushlash va boshqarish.

try {
  const data = await loginUser(email, password);
  message.success('Muvaffaqiyatli!');
} catch (err) {
  message.error(err.message); // Xato xabarini ko'rsatish
} finally {
  setLoading(false); // Har doim bajariladi
}
API so'rovlarda xato bo'lishi mumkin - try/catch bilan foydalanuvchiga xabar beramiz.
14CORS xatosi nima va qanday hal qilinadi? +

CORS - Cross-Origin Resource Sharing. Brauzer xavfsizligi - boshqa domendan so'rov yuborishda.

Xato ko'rinishi:

Access to fetch has been blocked by CORS policy

Yechim: Backend'da CORS ruxsat berish:

# FastAPI (Python):
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # Barcha domenlar
    allow_methods=["*"],
    allow_headers=["*"],
)
15HTTP metodlari (GET, POST, PUT, DELETE) farqi nima? +
  • GET - ma'lumot olish (foydalanuvchilar ro'yxati)
  • POST - yangi ma'lumot yaratish (ro'yxatdan o'tish)
  • PUT - mavjud ma'lumotni yangilash (profilni o'zgartirish)
  • DELETE - ma'lumotni o'chirish
// GET - foydalanuvchilarni olish
await fetch('/users');

// POST - yangi foydalanuvchi
await fetch('/users', { method: 'POST', body: {...} });

// DELETE - o'chirish
await fetch('/users/5', { method: 'DELETE' });

17Form va Form.Item qanday ishlaydi? +

Form - forma yaratish va validatsiya qilish.

<Form onFinish={handleSubmit}>
  <Form.Item
    name="email"

    <Input placeholder="Email" />
  </Form.Item>
  
  <Button htmlType="submit">Yuborish</Button>
</Form>

onFinish - validatsiya muvaffaqiyatli bo'lganda chaqiriladi.

18message.success va message.error nima qiladi? +

message - ekranning yuqorisida qisqa xabar ko'rsatish.

import { message } from 'antd';

// Muvaffaqiyat xabari (yashil):

// Xato xabari (qizil):

// Ogohlantirish (sariq):

19Button komponenti loading prop'i nima qiladi? +

loading - tugmada yuklanish animatsiyasini ko'rsatadi va tugmani disabled qiladi.

const [loading, setLoading] = useState(false);

  setLoading(true);  // Yuklanish boshlanadi

    await loginUser();
  } finally {
    setLoading(false); // Yuklanish tugaydi

Bu foydalanuvchini ikki marta bosishdan saqlaydi!

localStorage - brauzerda ma'lumotlarni saqlash. Sahifa yopilsa ham saqlanib qoladi.

21Arrow function ( => ) oddiy function'dan farqi nima? +

Arrow function - qisqaroq sintaksis:

// Oddiy function:
function add(a, b) {
  return a + b;
}

// Arrow function:
const add = (a, b) => a + b;

// Bir parametr:

// Ko'p qator:
const handleSubmit = async (values) => {

23Spread operator (...) nima qiladi? +

Spread operator - massiv yoki ob'ektni "yoyish".

// Ob'ektlarni birlashtirish:
const config = {
  headers: getAuthHeaders(),
  ...options,  // options ichidagi barcha kalitlar
};

// Massivga element qo'shish:
const newArray = [...oldArray, newItem];

// Ob'ektni nusxalash:

24import va export qanday ishlaydi? +

export - boshqa fayllarga ulashish:

// Named export:

export const getAuthHeaders = () => {...};

// Default export:
export default function LoginPage() {...}

import - boshqa fayldan olish:

// Named import (qavslar bilan):
import { API_URL, getAuthHeaders } from './client';

// Default import (qavssiz):
import LoginPage from './LoginPage';

// Aralash:
import App, { API_URL } from './App';
25Ternary operator ( ? : ) nima? +

Ternary operator - qisqa if/else:

// if/else:
if (loading) {
  return <Spin />;
} else {
  return <Content />;
}

// Ternary:
return loading ? <Spin /> : <Content />;

// JSX ichida:
<Button type={isActive ? "primary" : "default"}>
  {loading ? "Yuklanmoqda..." : "Kirish"}
</Button>

37inline style qanday yoziladi? +

  color: 'red',
  fontSize: '16px',  // font-size emas!

43map() metodi nima qiladi? +

map() - massivning har bir elementi uchun funksiya bajarib, yangi massiv qaytaradi.

const numbers = [1, 2, 3];

// doubled = [2, 4, 6]

  <ul>
    {users.map(user => <li key={user}>{user}</li>)}
  </ul>
);
React'da map() ishlatganda har bir elementga unique key berish kerak!
44filter() metodi nima qiladi? +

filter() - shartga mos elementlarni tanlab, yangi massiv qaytaradi.

const numbers = [1, 2, 3, 4, 5];
const even = numbers.filter(n => n % 2 === 0);
// even = [2, 4]

// Foydalanuvchilarni filtrlash:
const users = [
  { name: 'Ali', role: 'student' },
  { name: 'Vali', role: 'teacher' }
];
const teachers = users.filter(u => u.role === 'teacher');
// teachers = [{ name: 'Vali', role: 'teacher' }]
45find() metodi nima qiladi? +

find() - shartga mos BIRINCHI elementni topadi.

const users = [
  { id: 1, name: 'Ali' },
  { id: 2, name: 'Vali' }
];

const user = users.find(u => u.id === 2);
// user = { id: 2, name: 'Vali' }

// Topilmasa undefined qaytadi:
const notFound = users.find(u => u.id === 99);
// notFound = undefined
46key prop nima uchun kerak? +

key - React uchun har bir elementni aniqlash (unique identifier).

// XATO - key yo'q:

// Index ishlatish (oxirgi variant):

key React'ga qaysi element o'zgarganini aniqlashga yordam beradi.

49Token nima va nima uchun kerak? +

50Bearer nima? +

"Bearer" so'zma-so'z "tashuvchi" degan ma'noni anglatadi - token tashuvchi.
